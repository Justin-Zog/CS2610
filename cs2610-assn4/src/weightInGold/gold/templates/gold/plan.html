Software Development Plan

Phase 0: Requirements and Specifications

Deliver

  1. This project will include 2 apps. One named `unitconv` which will take care of the API related details of the website. The other named `gold`. `gold` will serve as a simple HTML web page.
  2. Gold will have a nav bar at the top that leads to plan.html (this file) and this plan will also contain a nav bar so you can jump between pages.
  3. The other view in `gold` that isn't plan will allow the user to find out how much their weight is worth in gold according to the latest market prices provided by the API.
  4. An important thing to note is that the entire page will not be loaded at the start. The API get request is called and then when it is fetched, the result will display on the page, without reloading it.
  5. This is done through an asynchronous GET request using fetch()
  6. The unitconv app will give us data from the API in the form of JSON. Which we then parse and use to our advatage.
  7. We must be able to convert between the US ton, Gram, Troy Oz (api gives us this one), kilogram, imperial pound, and ounce.

Phase 1: Design

Deliver

  • This assignment requires 2 apis. One gets us the Nasdaq data for the weight of Gold in dollars per Troy Ounces. The other is a Django Model that stores different conversion rates.
  • The model will require unit conversion between Tons, grams, troy ounces, kilograms, imperial pound, and ounce. It can convert between any and all units of measurement (both ways)
  • It will do this by making a fetch request to the unitconv django app.
  • This app will consist of a single page (kinda two pages since you can navigate to the unitconv page and see the json results).
  • This single page will allow the user to input their weight and then a new div will appear without refreshing the page This new div will contain a timestamp as well as how much XXX grams/tons/whatever of gold is worth.
  • Other than that there isn't much else to this app. Just checking the users input (never trust those dudes) and making sure they know if the api fetch() failed.
  • For the API that I'll be writing in Django. I used this link for conversion factors. Wolfram Alpha
