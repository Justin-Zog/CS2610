Phase 0: Requirements and Specifications

Deliver

  1. Write an app whose client side is written in JavaScript and a Django server side component written in Python.
  2. This app will have 6 webpages served as Django templates. Each page must be clearly identified by a title and a heading. Each page is also required to have a common navigation section.
  3. I will use Erik's CSS file to style all the pages as this helps with simplicity.
  4. The first page should be the index. It serves as a calculator and has 3 main sections. The first of which creates a mathematical expression. The second will show expressions that have been computed, and the third will show a handful of (about 5) saved expressions. Those saved expressions are stored on the database.
  5. The next page will be the Recent Expression page. It is essentially the save as section 3 of the calculator page except it shows all saved expressions.
  6. The next page is the Undefined Expressions page. It will display any expressions with undefined results such as dividing by 0. It is essentially a specialized version of the second page.
  7. The next page is the disagreeing results page. It consists of any saved expression where the javascript math's answer differs from python's math.
  8. The next page shows all the expressions divided into sections based on which operator was used for the equation.
  9. The last page will be this one. The software development plan page.
  • For this app we also have to do some confirmation that the client is not doing something stupid. Which as we all know, happens quite frequently.
  • To do this I will test if an input is empty, if the inputs are actually numbers, and if the operator is included in my operator table. If any if these tests go wonky an error message will be produced and the user won't be able to save their equation.
  • On top of validated the content from the user on the client side, I will also confirm that it is valid on the server side. If something goes wrong on the server side, the user will be redirected to an error page.

Phase 1: Design

Deliver

  • Most the functionality of this program lies in the POST requests and validating that input is correct.
  • There will be a function that compares the javascript results with the python results to see if they are the same. This python will also double check that the input is valid on the server side. This is an extra measure of security to stop malware or something bad from happening.
  • There will have to be some type of function that populates the operator table with approved arithmetic operators. A migration script will be used for this.
  • On the server side there will be some behind the scenes magic going on, this magic is basically filtering the saved expression by different means and putting them on the page.
