Phase 0: Requirements and Specifications

Deliver

  1. Unlike previous projects, this one will not use any Django frameworks!
  2. We will just be using 4 files: plan.html, index.html, script,js, style.css. Because we are using Vue.js this is possible.
  3. This project must use Vue.js front-end framework to dynamically build a webpage that users can interact with.
  4. It will consist of a single page that will use 2 different API's to get data and display them on the screen. The user will then be able to interact with that content.
  5. One of the API's, Abstract API, will get an geolocation (latitude and longitude) based on the user's IP address and display it on the screen. Once we have that data, we can do the 2nd API call.
  6. The next API, OpenWeatherMap, provides weather information. This project will use the previously obtained latitude and longitude to fetch the current weather data and display it to the screen.
  7. The rest of this project has to do with manipulating the DOM using Vue.js. This program will take advantage of Vue.js to dynamically change the webpage without making any fetch requests or reloading the page.
  8. IMPORTANT: Despite the user being able to vote by clicking divs, the app does not need to be persistent.

Phase 1: Design

Deliver

  • This is a fairly simple app. I will describe the divs needed below.
  • Header: This div simply contains the name of the app
  • Your Location: This div uses Abstract API to get the users location using their IP addres, then displays that information.
    Before the information is obtained, a loading prompt lives inside the div.
  • Current Conditions: This div displays the current weather conditions as obtained by OpenWeatherMap's API.
    Before this information is fetched a loading prompt lives in the div.
  • 5 day 3 hour forecast: This div displays the information for the weather prediction for the next 5 days in 3 hour increments.
    Each one of the predictions is stored in it's own div within this one. Before the data is fetched a loading message is present.
    Each div is clickable and will change between 3 different background colors. Green (likely), Red (unlikely), Gray (neutral).
    This makes our app user interactable. Although we allow the user to change these divs we do not persist the data to a database.

Methods

Most methods required just use classic javascript to fetch data from an API using promises and .then() methods.

The other methods needed come from Vue.js. We can use Vue within our html by adding Vue 'properties' to the element. An example of this would be the Vue 'if' statement. v-if="awesome". We can find all the methods we need in Vue's documentation.

Phase 3: Testing & Debugging

Deliver

While testing and debugging I found that one of the API's only allows one API call a second so if you spam refreshing the page the API breaks.
The API convieniently gives an 'undefined' datatype instead of actual json data. I let the user know they are refreshing the page too quickly.

I also tested to see if my app would dynamically respond to the APIs being down and it does.
It also lets the user know that the app is retrieving data and to be patient if the connection is throttled.
